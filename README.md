# CryptoSentry / 币哨

加密货币实时监控哨兵——基于币安API，多指标共振分析，异动即时推送。

## 功能特性

- **实时行情**：通过币安WebSocket实时获取K线数据
- **价格日志**：定时输出监控币种的实时价格与涨跌幅
- **行情异动报警**：价格相对当根K线开盘价涨跌超过阈值时，钉钉推送通知
- **整数关口告警**：价格穿越整数关口（如$76,000）时推送通知，同一关口5分钟内不重复
- **指标状态报告**：每60秒输出所有指标的当前值及距告警阈值的差距，随时掌握盘面
- **技术指标**：8种主流指标，插件式架构，可自由扩展
  - RSI（相对强弱指数）
  - MACD（移动平均收敛散度）
  - 布林带（Bollinger Bands）
  - EMA（指数移动平均线）
  - KDJ（随机指标）
  - ATR（真实波幅）
  - SuperTrend（趋势跟踪）
  - OBV（能量潮/量价背离）
- **信号系统**：2个以上指标同方向共振才告警，单指标触发只记录不推送；信号强度评级（弱/中/强），冷却防重复
- **钉钉通知**：加签安全验证，Markdown格式消息，发送频控
- **数据存储**：SQLite存储K线历史和信号记录

## 快速开始

### 1. 安装依赖

```bash
pip3 install -r requirements.txt
```

### 2. 修改配置

编辑 `config.yaml`：

```yaml
proxy:
  enabled: true
  http: "http://127.0.0.1:7897"    # 你的代理地址
  https: "http://127.0.0.1:7897"

dingtalk:
  webhook: "https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN"
  secret: "YOUR_SECRET"
```

### 3. 测试连接

```bash
python3 main.py --test
```

### 4. 启动监控

```bash
# 前台运行
python3 main.py

# 后台运行
python3 main.py --daemon
```

## 告警规则

### 核心规则：多指标共振

**2个以上指标同方向才会触发告警推送**，单指标触发只记录日志不推送。

共振强度评级：
- **强**：3个以上指标同方向，或2个指标中包含强信号
- **中**：2个中等强度指标同方向
- **弱**：仅1个指标触发（不推送）

### K线周期

每个K线周期（1h/4h/1d）**独立计算、独立告警**。也就是说1h收盘算一次、4h收盘算一次、1d收盘算一次，互不干扰。告警消息会标注K线周期。

### 频控机制

| 频控维度 | 说明 | 默认值 |
|----------|------|--------|
| 信号冷却 | 同一币种同一方向的信号冷却时间 | 15分钟 |
| 每小时上限 | 每币种每小时最大通知数 | 10次 |
| 钉钉频控 | 两次钉钉消息最小间隔 | 20秒 |
| 整数关口冷却 | 同一关口穿越告警冷却时间 | 5分钟 |
| 行情异动冷却 | 同币种异动报警冷却时间 | 10分钟 |

## 技术指标详解

### RSI（相对强弱指数）

衡量价格变动的速度和幅度，判断超买超卖状态。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `period` | 计算周期 | 14 |
| `overbought` | 超买线 | 70 |
| `oversold` | 超卖线 | 30 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | RSI < 30（进入超卖） | RSI<20=强，否则=中 |
| 卖出 | RSI > 70（进入超买） | RSI>80=强，否则=中 |
| 买入 | RSI从上方跌破30 | 中 |
| 卖出 | RSI从下方突破70 | 中 |

**指标状态报告显示**：`RSI=45.2 (距超买24.8/距超卖15.2)`

---

### MACD（移动平均收敛散度）

通过快慢均线的差值判断趋势方向和动量变化，金叉死叉是经典交易信号。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `fast` | 快线周期 | 12 |
| `slow` | 慢线周期 | 26 |
| `signal` | 信号线周期 | 9 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | DIF上穿DEA（金叉） | 强 |
| 卖出 | DIF下穿DEA（死叉） | 强 |
| 买入 | 柱线从负转正 | 中 |
| 卖出 | 柱线从正转负 | 中 |

**指标状态报告显示**：`MACD: DIF=120.5 DEA=115.3 柱=5.2 金叉区(距交叉5.2)`

---

### 布林带（Bollinger Bands）

由中轨（均线）和上下轨（±N倍标准差）构成，价格触及上下轨意味着偏离常态。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `period` | 中轨周期 | 20 |
| `std` | 标准差倍数 | 2.0 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | 价格触及下轨 | 低于下轨=强，等于=中 |
| 卖出 | 价格触及上轨 | 高于上轨=强，等于=中 |

**指标状态报告显示**：`布林: 上=78500 下=74200 位置=42.1%(距上2494/距下1806)`

---

### EMA（指数移动平均线）

对近期价格赋予更高权重，比简单均线更灵敏。通过不同周期EMA的交叉判断趋势。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `periods` | EMA周期列表 | [7, 25, 99] |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | EMA7上穿EMA25（短期上穿中期） | 强 |
| 卖出 | EMA7下穿EMA25（短期下穿中期） | 强 |

**指标状态报告显示**：`EMA7=76200(下方) | EMA25=76800(下方) | EMA99=75500(上方)`

---

### KDJ（随机指标）

通过K、D、J三条线判断超买超卖和金叉死叉，比RSI更灵敏。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `k_period` | K值周期 | 9 |
| `d_period` | D值平滑周期 | 3 |
| `j_period` | J值平滑周期 | 3 |
| `overbought` | 超买线 | 80 |
| `oversold` | 超卖线 | 20 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | K线低位金叉（K<D后K>D，且K<20） | 强 |
| 卖出 | K线高位死叉（K>D后K<D，且K>80） | 强 |
| 买入 | K < 20（进入超卖） | 中 |
| 卖出 | K > 80（进入超买） | 中 |

**指标状态报告显示**：`KDJ: K=52.3 D=48.1 J=60.7 ⚠️超买`

---

### SuperTrend（趋势跟踪指标）

基于ATR的趋势判断指标，方向翻转意味着趋势反转。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `period` | ATR计算周期 | 10 |
| `multiplier` | ATR乘数 | 3.0 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | 方向从空头翻为多头 | 强 |
| 卖出 | 方向从多头翻为空头 | 强 |

**指标状态报告显示**：`SuperTrend: 多头🟢 趋势线=74800(距1206)`

---

### ATR（真实波幅）

衡量市场波动程度，不判断方向，只判断波动大小。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `period` | 计算周期 | 14 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | ATR/价格 < 1%（波动极低，可能酝酿突破） | 弱 |
| 卖出 | ATR/价格 > 5%（波动异常高） | 弱 |

**指标状态报告显示**：`ATR=850.5 (1.12%)`

---

### OBV（能量潮/量价背离）

通过成交量变化判断价格趋势的可靠性，量价背离意味着趋势可能反转。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `divergence_lookback` | 背离回看周期 | 20 |

**信号触发条件**：

| 信号 | 条件 | 强度 |
|------|------|------|
| 买入 | 量价底背离（价格下跌但OBV上升） | 中 |
| 卖出 | 量价顶背离（价格上涨但OBV下降） | 中 |

**指标状态报告显示**：`OBV=1,234,567,890`

## 配置说明

### 代理

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `proxy.enabled` | 是否启用代理 | `true` |
| `proxy.http` | HTTP代理地址 | - |
| `proxy.https` | HTTPS代理地址 | - |

### 交易对

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `symbols[].name` | 币种名称 | - |
| `symbols[].intervals` | K线周期 | `["1h", "4h", "1d"]` |
| `symbols[].price_alerts` | 价格突破提醒价位列表 | - |

### 价格日志

定时在控制台输出所有监控币种的实时价格和1h涨跌幅。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `price_log.enabled` | 是否启用 | `true` |
| `price_log.interval_seconds` | 输出间隔（秒） | `10` |

### 行情异动报警

当价格相对当根1h K线开盘价涨跌超过阈值时，发送钉钉通知并记录日志。同一币种在冷却时间内不重复报警。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `price_change_alert.enabled` | 是否启用 | `true` |
| `price_change_alert.pct_threshold` | 涨跌幅阈值（%） | `1.0` |
| `price_change_alert.cooldown_seconds` | 同币种报警冷却时间（秒） | `600`（10分钟） |

### 整数关口告警

价格穿越整数关口（如$76,000）时推送通知。同一关口不管突破还是跌破，冷却时间内不重复告警，避免价格在关口附近来回震荡时频繁打扰。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `round_price_alert.enabled` | 是否启用 | `true` |
| `round_price_alert.step` | 整数关口步长 | `1000` |
| `round_price_alert.cooldown_seconds` | 同一关口冷却时间（秒） | `300`（5分钟） |

### 指标状态报告

定时输出所有启用指标的当前值及距告警阈值的差距，让你随时掌握盘面状态，不用等到告警才知道指标走到哪了。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `indicator_report.enabled` | 是否启用 | `true` |
| `indicator_report.interval_seconds` | 报告间隔（秒） | `60` |

报告示例：
```
📈 BTCUSDT $76,005.65 [1h]
  RSI=45.2 (距超买24.8/距超卖15.2)
  MACD: DIF=120.5 DEA=115.3 柱=5.2 金叉区(距交叉5.2)
  布林: 上=78500 下=74200 位置=42.1%(距上2494/距下1806)
  KDJ: K=52.3 D=48.1 J=60.7
  EMA7=76200(下方) | EMA25=76800(下方) | EMA99=75500(上方)
  SuperTrend: 多头🟢 趋势线=74800(距1206)
  ATR=850.5 (1.12%)
  OBV=1,234,567,890
```

### 信号系统

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `signal.cooldown_minutes` | 同币种同方向信号冷却时间（分钟） | `15` |
| `signal.max_notifications_per_hour` | 每币种每小时最大通知数 | `10` |
| `signal.min_strength` | 最低信号强度（`weak`/`medium`/`strong`） | `"medium"` |

### 钉钉通知

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `dingtalk.webhook` | 机器人Webhook地址 | - |
| `dingtalk.secret` | 加签密钥 | - |
| `dingtalk.message_type` | 消息格式（`markdown`/`text`） | `markdown` |
| `dingtalk.min_interval_seconds` | 发送频控，两次消息最小间隔（秒） | `20` |

## 新增指标

1. 在 `core/indicators/` 下创建新文件
2. 继承 `BaseIndicator`，实现 `calculate()` 和 `get_signals()`
3. 在 `core/indicators/__init__.py` 注册
4. 在 `config.yaml` 中添加配置并 `enabled: true`
